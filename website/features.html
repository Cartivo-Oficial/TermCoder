Features

A terminal agent with real tools.

Everything the terminal-first crowd wants, wrapped in a keyboard-driven interface that stays out of the way — and a study mode for everyone else.

// free

Free, with no API key

Open a folder and start — TermCoder ships with a free, keyless model, so there's no sign-up, card, or quota to configure. When the free tier is busy it retries and falls back on its own, and /upgrade connects a free Gemini key in two steps when you want more.

// terminal-first

Built for the terminal

Slash commands, @file mentions to pull code into context, a live model picker, multi-line input with word-jump editing, and a trust prompt the first time you open a folder. Nothing leaves the keyboard.

// any model

Any model, or fully local

Bring Anthropic, OpenAI, or Google Gemini with your own key, or point it at a local model through Ollama and run with no account and no data leaving your machine. Switch models mid-session with /model.

// modes

Build and Plan modes

Plan mode reads the code and proposes an approach without touching a single file; Build mode carries the change out, asking permission before anything that writes or runs. Toggle with shift+tab.

// termcoder/auto

A brain that reviews itself

The default termcoder/auto routes each task by difficulty — a fast model for simple edits, a stronger one for hard work — then reviews its own diff before finishing and fixes what it flags.

// autonomous

Works unattended

Hand it a goal with /background and it plans, edits, and runs your project's tests, feeding failures back and fixing until the check passes or it runs out of rounds. Every round is checkpointed, so you can revert.

// specialists

Sub-agents and skills

Hand focused work to specialist sub-agents — reviewer, tester, debugger, architect — with the $ key. Reusable skill playbooks load on demand, so idle skills cost nothing.

// github backbone

Sync, share, and packs

Sync your settings, drafts, and study decks across machines through a private gist. Share a session as a link anyone can open. Bundle your agents, skills, and commands into a pack, and install someone else's with /pack install — all on GitHub, no server to run.

// classrooms

Classrooms for teachers and students

A teacher creates a class, students join with a link, and assignments and submissions ride GitHub comments. Shared packs install on join, so everyone starts with the same agents and skills. Async by design — nothing to host.

// anywhere

Terminal, desktop, or browser

Run the CLI, the desktop app, or open the full interface in a browser from a local termcoder serve — the same engine everywhere. Shared sessions open in a hosted viewer with just the link.

// token economy

Kind to your context

Old tool output is elided from the model's context and long results are capped, so a long session stops re-billing everything it has already read. A context meter shows you how full the window is.

// study mode

A study companion inside

Switch to the termexplorer model and the same tool becomes a patient tutor for schoolwork — explaining, summarizing, and building flashcards. See study mode →
