01
Start · no account

No API key.

It opens on a keyless, community-hosted model. No sign-up, no card, nothing to configure.

Two caveats we would rather state than hide: that model is rate-limited when busy, and prompts go to a third party we do not run. Point it at a local Ollama model to make it private.

term — first run
❯ term
● termcoderfree/auto  free · no key
❯ fix the flaky test in date.js
✓ read date.test.js
✓ edit date.js
✓ npm test  → passing

Models

Or bring your own.

termcoder/auto sends each turn to the right model — a fast one to read a file, a stronger one to reason about a bug. A router and a prompt layer, not a model we trained.

04
Memory

It remembers.

Tell it once that you use pnpm, that the auth module is fragile, that you hate barrel files. It keeps that across sessions, so you stop re-explaining your own project every morning.

term
❯ /remember pnpm, never npm
✓ saved to memory

# a week later, unprompted
❯ add zod as a dependency
✓ pnpm add zod  (remembered)
05
Retrieval

It reads first.

A symbol index and a repo map let it find where a thing is defined and how the project fits together — before it answers, instead of guessing from whatever file you had open.

06
Desktop

A real shell, inside the app.

Chat, an editor, and an embedded terminal in one window. It scans your PATH and drops a one-click chip on every coding CLI it finds — Claude Code, Codex, Gemini CLI. The shell keeps running while you chat.

07
Autonomous

Give it a goal.

Hand it a task and a verify command. It works, runs the command, reads the failure, and goes again — until it passes. Every turn is checkpointed, so you can walk it back.

term
❯ /background make the build pass
● round 1 → npm run build
  ✗ 2 type errors
● round 2 → fixed, ran again
  ✓ build passed
