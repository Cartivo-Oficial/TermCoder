01
Start · no account

No API key.

It opens on a keyless, community-hosted model. No sign-up, no card, nothing to configure. Type a task and it works.

Two things you should know, because we would rather say them than hide them: that model is rate-limited when it is busy, and your prompts go to a third party we do not run. Point it at a local Ollama model when you want it private and unlimited.

02
Models

Or bring your own.

termcoder/auto routes each turn to the best model for the task — a fast one to read a file, a stronger one to reason about a bug. It is a router and a prompt layer, not a model we trained.

03
Sign in · experimental

Use the plan you pay for.

Sign in with an existing Claude Pro/Max or ChatGPT Plus/Pro subscription instead of buying API credits. This is experimental: it rides the vendors' own login flows, and those can change without warning.

04
Memory

It remembers.

Tell it once that you use pnpm, that the auth module is fragile, that you hate barrel files. It keeps that across sessions, so you stop re-explaining your own project every morning.

05
Retrieval

It reads first.

A symbol index and a repo map let it find where a thing is defined and how the project fits together — before it answers, instead of guessing from whatever file you had open.

06
Desktop

A real shell, inside.

The desktop app embeds a terminal. It scans your PATH and puts a one-click chip on every coding CLI it finds — Claude Code, Codex, Gemini CLI. The shell keeps running while you chat.

07
Autonomous

Give it a goal.

Hand it a task and a verify command. It works, runs the command, reads the failure, and goes again — until the command passes. Every turn is checkpointed, so you can walk it back.

08
TermExplorer

A tutor is built in.

The same engine, pointed at studying instead of shipping. Spaced-repetition flashcards, practice quizzes, worked homework, and a streak that survives your weekends.
