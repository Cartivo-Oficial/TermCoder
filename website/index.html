01
Start · no account

No API key.

It opens on a keyless, community-hosted model. Two caveats we would rather state than hide: it is rate-limited when busy, and prompts go to a third party we do not run. Point it at a local Ollama model to make it private and unlimited.

02
Models

Or bring your own.

termcoder/auto sends each turn to the right model — a fast one to read, a stronger one to reason. It is a router and a prompt layer, not a model we trained. Anthropic, OpenAI, Google, and local Ollama.

read a file→gemini-flash
reason about a bug→claude-sonnet
write & run tests→gpt

04
Memory

It remembers.

Tell it once — your stack, the fragile module, the thing you always forget. It keeps that across sessions, so you stop re-explaining your own project every morning.

~/.termcoder/memorypersists across sessions
  • uses pnpm, never npm
  • the auth module is fragile — tread carefully
  • no barrel files
  • tests run with npm test, not a watcher
05
Retrieval

It reads first.

A symbol index and a repo map let it find where a thing is defined and how the project fits together — before it answers, instead of guessing from whatever file you had open.

06
Desktop

A real shell, inside the app.

Chat, an editor, and an embedded terminal in one window. It scans your PATH and drops a one-click chip on every coding CLI it finds. The shell keeps running while you chat.

07
Autonomous

Give it a goal.

Hand it a task and a verify command. It works, runs the command, reads the failure, and goes again — until it passes. Every turn is checkpointed, so you can walk it back.

round 1 edit → npm run build✗ 2 type errors
round 2 fix types → build✗ 1 test failing
round 3 fix test → build✓ build passed
