Why termcoder

Everything the terminal-first crowd wants — and a study mode for everyone else.

An agent loop with real tools, wrapped in a CLI that's a pleasure to live in.

// terminal-first

Built for the terminal

Slash commands, @file mentions, a live model picker, multi-line input, sub-agents — all keyboard-driven, no window required.

// two brains

termcoder/auto + termexplorer

An orchestrator that routes by task difficulty and self-reviews its changes — plus a sister study persona for schoolwork.

// free & local

Runs for free

Point it at a local Ollama model, or use a free Gemini key. No account, no lock-in. Bring Anthropic or OpenAI if you want.

// token economy

Kind to your tokens

Old tool output is elided from context and long results are capped, so a long session stops re-billing everything it's read.

// specialists

Skills & sub-agents

Reusable skill playbooks load on demand; specialist sub-agents (reviewer, tester, debugger, architect) take focused sub-tasks.

// craft

An interface you'll enjoy

Animated starfield home, a gradient wordmark, syntax-highlighted diffs, favorites, and a folder-trust prompt on first run.

📚
termexplorer

A study buddy in the same tool.

Switch to the termexplorer model and termcoder becomes a patient tutor: it explains step by step, summarizes your notes, helps with homework by showing the working, and builds flashcards, practice quizzes, and study plans. Built for students — no programming needed.

Install

Two ways to run it.

The CLI installs from npm and works everywhere. The desktop app is a download.

Terminal (CLI)

Install once, then type term in any project folder.

# PowerShell or CMD
npm install -g @termcoder​/tui
# then, in any folder:
term

Need a model? Run /setup inside the app — it walks you through a free option.

Desktop app

A window with tabs, inline diffs, voice, and a simplified study mode — same engine as the CLI.
